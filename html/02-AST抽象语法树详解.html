2.1

什么是 AST？抽象语法树的本质

AST（Abstract Syntax Tree）是源代码的树形结构化表示。"抽象"意味着它省略语法细节（分号、括号、空格），只保留程序结构的本质信息。

cst-vs-ast.txt · 具体语法树 vs 抽象语法树
// 表达式: 3 + 4 * 5 // CST (Concrete Syntax Tree) — 包含所有语法细节： Expression / | \ Number '+' Expression | / | \ 3 Number '*' Number | | 4 5 // AST (Abstract Syntax Tree) — 去除冗余语法： (+) / \ 3 (*) / \ 4 5
2.2

编译前端流水线词法分析与语法分析

词法分析 (Lexing)
  • 源码 → Token 流
  • 核心技术：正则表达式、DFA、手动编码
  • 难点：歧义消除（如 JS 中 / 是除法还是正则？）
  • 难点：向前看（Lookahead）
语法分析 (Parsing)
  • Token 流 → AST
  • 自顶向下：递归下降（最常用）、LL(k)、Packrat
  • 自底向上：LR(k)、LALR(1)、GLR、Earley
  • 递归下降是工业界最主流选择
lexing-example.js · 词法分析输出
// 输入: const x = 42 + y; // Token 流: [ { type: 'Keyword', value: 'const', pos: 0 }, { type: 'Identifier', value: 'x', pos: 6 }, { type: 'Punctuator', value: '=', pos: 8 }, { type: 'Numeric', value: '42', pos: 10 }, { type: 'Punctuator', value: '+', pos: 13 }, { type: 'Identifier', value: 'y', pos: 15 }, { type: 'Punctuator', value: ';', pos: 16 }, ]

2.3 → 2.4 · 核心模式
Vistor 模式 — AST 遍历的通用设计模式
Enter vs Leave：进入节点时和离开节点时的两个回调时机。深度优先遍历保证子节点在父节点的 Enter/Leave 之间被完全处理。
2.4

Visitor 遍历模式Enter / Leave 与 Path

traversal-order.txt · 深度优先遍历顺序
// AST 结构: Program / \ Variable Function Declaration Declaration | / \ let Param Body | | x return x // Enter 顺序（进入节点时）: Program → VariableDeclaration → let → FunctionDeclaration → Param → x → Body → return → x // Leave 顺序（离开节点时，逆序）: x ← return ← Body ← x ← Param ← FunctionDeclaration ← let ← VariableDeclaration ← Program
visit() 回调
  • 按节点类型注册处理器
  • 进入节点时调用
  • 离开节点时调用（子节点已处理完）
Path 对象（Babel）
  • 包装节点，提供上下文（父节点、作用域）
  • replaceWith() 替换节点
  • remove() 删除节点
  • skip() / stop() 控制遍历

2.6

AST 核心应用编译 · 格式化 · Linting · 安全

应用原理工具
代码转换解析 AST → 遍历修改 → 代码生成Babel, SWC, tsc
代码格式化解析 AST → 丢弃原始空白 → 按规则重新生成Prettier, Black, gofmt
Linting遍历 AST → 检查规则 → 报告问题ESLint, Pylint, Clippy
复杂度分析遍历 AST → 统计分支节点圈复杂度、认知复杂度
依赖分析提取 ImportDeclaration → 构建依赖图depcheck, madge
安全检测检测危险模式（eval, innerHTML）Semgrep, Bandit
⚡ 高级主题：容错解析与增量解析
  • 容错解析：tree-sitter 能处理不完整代码（IDE 场景），标记 ERROR 节点和 MISSING 部分
  • 增量解析：只重新解析修改部分，复用其余 AST。tree-sitter 核心创新
  • Chomsky 层级：正则文法(Type 3) → Token 级，上下文无关文法(Type 2) → 语法级
